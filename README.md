# 배송알리미

배송알리미는 내 휴대폰에 들어오는 배송 관련 알림을 모아, 주문/배송/완료 상태를 한곳에서 확인하기 위한 로컬 전용 Android 앱입니다.

Supabase나 별도 서버를 사용하지 않고, 앱 로그인도 없습니다. 배송 정보와 모니터링 설정은 사용자의 기기 안에 저장됩니다.

## 이런 용도로 사용합니다

- 여러 쇼핑몰과 택배사의 배송 상태를 한 화면에서 확인
- 오늘 기준 주문 내역, 배송 중, 오늘 완료, 오늘 도착 예정 배송 정리
- 업체별, 일별, 월별 배송 현황 확인
- 운송장번호를 알고 있는 배송을 직접 등록
- 카카오톡 알림톡, SMS, Gmail 등 앱 알림으로 들어오는 배송 정보를 자동 수집
- 카드 결제 승인 문자·쇼핑몰 주문완료 알림도 "주문했다"는 기록으로 자동 등록
- 스마트택배 API로 배송 상태를 직접 조회 (선택 사항)
- 앱을 열지 않아도 백그라운드에서 알림을 확인해 자동 등록

## 화면 구성

하단 메뉴는 아이콘 중심으로 구성되어 있습니다.

- 업체별: 택배사 또는 쇼핑몰 기준으로 배송을 확인합니다.
- 일별: 오늘 기준 배송 현황을 주문 내역, 배송 내역, 오늘 완료, 오늘 배달 예정으로 나누어 보여줍니다.
- 월별: 월간 캘린더 형태로 배송 흐름을 봅니다.
- 필터: 업체와 배송 상태를 각각 여러 개 선택하고 날짜 기간을 지정해 목록을 좁힙니다. 적용하면 일별 화면과 같은 형식(주문/배송중/배달완료/집하 건수 카드 + 상세 목록)으로 결과가 나옵니다.
- 설정: 다크/라이트 테마, 업체 목록(직접 추가/삭제), 알림 접근·카카오톡 접근성 권한, 스마트택배 조회 API 키, 피싱 의심 보관함, 테스트 기능을 관리합니다.
- 사용자: Gmail, 기타 이메일, SMS, 카카오톡, 텔레그램, WhatsApp 같은 모니터링 소스를 관리합니다.

### 업체 목록

설정 > 로컬 화면의 업체 목록에서 배송 조회에 사용할 택배사를 직접 추가하거나 지울 수 있습니다. CJ대한통운/한진택배/롯데택배/우체국택배/로젠택배처럼 미리 등록된 업체는 실제 운송장번호 형식과 조회 코드를 그대로 쓰고, 새로 추가한 업체는 이름으로 문자를 인식하고 일반적인 숫자 운송장번호 형식을 사용합니다. 이미 등록된 배송이 있는 업체는 목록에서 지워도 자동으로 다시 추가됩니다.

### 카드 결제 · 쇼핑몰 주문완료 알림

카드사에서 오는 결제 승인 문자/알림톡이나 쇼핑몰의 "주문완료" 알림은 아직 배송이 시작된 게 아니라 결제/주문만 일어난 상태이므로, "주문했다"는 사실만 기록하고 배송처럼 여러 날에 걸쳐 이월시키지 않습니다.

- 캘린더에는 알림을 받은 그날 하루에만 표시됩니다. 다음 날부터는 나타나지 않습니다.
- 업체별/일별/월별의 진행중·완료 목록에는 아예 나타나지 않습니다 (배송이 아니므로).
- 나중에 같은 물건의 실제 배송 알림(운송장번호 포함)이 오면, 그 배송은 해당 택배사 아래 새 항목으로 별도 등록됩니다.

### 스마트택배 API 조회

설정 > 로컬 화면에서 스마트택배(스윗트래커) API 키를 등록하면, 배송 목록과 상세 화면에서 `배송 조회`/`전체 배송 조회` 버튼으로 최신 상태를 직접 조회할 수 있습니다. 키가 없어도 알림 기반 자동 추적은 그대로 동작하며, 무료 키는 하루 약 100건까지만 조회됩니다.

### 피싱 의심 보관함

배송을 사칭한 피싱 문자/알림으로 의심되는 메시지는 배송 목록에 등록되지 않고 설정 > `피싱 의심 보관함`에 따로 보관됩니다. 자동으로 삭제되지 않으므로, 오탐으로 보관된 경우 직접 확인 후 지울 수 있습니다.

### 백그라운드 자동 동기화

앱을 열어두지 않아도 배송 관련 알림이 오면 백그라운드에서 자동으로 확인해 등록합니다. 새로 등록된 내용이 있으면 앱 아이콘에 배지(숫자)로 표시됩니다.

## 처음 사용할 때

1. APK를 설치합니다.
2. 앱을 열고 설정 화면으로 이동합니다.
3. 로컬 탭에서 알림 접근 권한 상태를 확인합니다.
4. 카카오톡 알림톡을 수집하려면 카카오톡 접근성 수집 설정으로 이동해 배송알리미 카카오톡 수집을 허용합니다.
5. 사용자 화면에서 사용할 모니터링 소스를 켭니다.
6. 필요한 경우 계정, 비밀번호, 토큰을 입력합니다.
7. 배송 목록의 `+` 버튼으로 직접 배송을 등록할 수 있습니다.

## 권한 안내

배송알리미는 Android 시스템 설정에서 사용자가 직접 허용한 범위 안에서만 동작합니다.

- 알림 접근 권한: 휴대폰에 올라온 알림을 배송 후보로 확인하기 위해 필요합니다.
- 접근성 권한: 카카오톡 알림톡 화면의 배송 관련 텍스트를 읽기 위해 필요합니다.

권한은 앱 안의 스위치만으로 완전히 켜지는 것이 아니라, Android 시스템 설정 화면에서 사용자가 직접 허용해야 합니다. 앱의 설정 화면은 해당 시스템 설정으로 이동할 수 있도록 도와줍니다.

## 데이터와 보안

- 배송 데이터는 로컬 SQLite 데이터베이스에 저장됩니다.
- Gmail, 기타 이메일, SNS 계정 정보는 Android secure storage에 암호화 저장됩니다.
- 앱 자체 로그인은 없습니다.
- 멀티 유저를 지원하지 않습니다.
- 서버 동기화, 클라우드 백업, 원격 저장은 현재 사용하지 않습니다.

## 테스트 기능

설정 화면의 테스트 영역에서는 개발과 검증을 위한 샘플 입력을 사용할 수 있습니다.

- Gmail 샘플 보내기 / SMS 샘플 보내기: 매번 택배사·상태·상품명·운송장번호를 무작위로 조합한 배송 샘플을 파서에 넣어 배송 등록 흐름을 확인합니다. 눌러도 항상 같은 내용이 아니라 매번 다른 배송이 등록됩니다.
- 알림 동기화 / 현재 알림 다시 스캔: 알림 리스너와 접근성 서비스가 캡처한 원문을 배송 정보로 변환합니다. 다시 스캔은 현재 떠 있는 알림도 다시 훑습니다.
- 알림 주입 테스트: 알림 본문을 직접 입력해 배송으로 인식되는지 확인합니다.

테스트 기능은 실제 발송 기능이 아니라 앱 내부 파싱과 저장 흐름을 검증하기 위한 기능입니다.

앱 밖에서 실제 알림을 발생시키는 테스트는 `tools/test_notify`를 사용합니다.

```powershell
cd tools\test_notify
python send_email.py --to gmail
python send_email.py --to naver
```

SMS는 Naver Cloud SENS 설정이 준비된 경우 `python send_sms.py`로 보낼 수 있습니다. 실제 계정/API 키는 `tools/test_notify/.env`에 넣고, 이 파일은 Git에 커밋되지 않습니다.

## 현재 한계

- Gmail/네이버 메일/SMS는 각 앱의 알림에 표시되는 제목과 본문을 기준으로 수집합니다. 메일함이나 문자 DB를 직접 읽지는 않습니다.
- 알림에 운송장번호나 배송 키워드가 충분히 표시되지 않으면 자동 등록되지 않을 수 있습니다.
- 카카오톡 수집은 접근성 기반이라 카카오톡 화면 구조 변경, 시스템 글꼴 크기, 알림톡 형식에 따라 추가 검증이 필요합니다.
- 같은 결제·주문이 SMS와 카카오톡 알림톡 등 서로 다른 채널로 각각 들어오면, 문구 차이 때문에 중복으로 등록될 수 있습니다. 이 경우 배송 목록에서 직접 지우면 됩니다.
- Play Store 배포를 전제로 하지 않고, 우선 APK 직접 설치 방식으로 사용합니다.

## 개발자 확인용

프로젝트는 Flutter 기반입니다.

```powershell
cd app
flutter analyze
flutter test
flutter build apk --debug
```

Android 기기에 기존 데이터를 유지하며 업데이트하려면 다음처럼 설치합니다.

```powershell
adb install -r build\app\outputs\flutter-apk\app-debug.apk
```

Windows 빌드는 Android 배포 전 화면과 동작을 확인하기 위한 테스트 용도입니다.

```powershell
flutter build windows --debug
```
